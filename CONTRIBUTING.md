# 🤝 Contributing to BranReadme

Thank you for considering contributing to **BranReadme** 🚀
This project aims to make building GitHub README files fast, simple, and powerful.

---

## Contribution Guidelines

To keep the project clean and maintainable, please follow these rules:

### Code Style

- Follow the existing **React + Vite + Tailwind CSS** structure
- Keep code clean, readable, and consistent
- Use meaningful variable and function names

### Commit Messages

- Use clear and structured commit messages:
  - `feat: add template favorites`
  - `fix: resolve preview rendering issue`
  - `refactor: improve drag-and-drop logic`

### Issues First

- For major features or changes, **open an issue first**
- Discuss the idea before starting work

### Pull Requests

- Keep PRs **small and focused**
- One feature or fix per pull request
- Add a clear description of changes

### Testing

- Test changes before submitting
- Ensure:
  - No breaking features
  - UI works correctly
  - Build runs successfully

### Security

- Never commit:
  - `.env` files
  - API keys
  - Sensitive data

### UI / UX Consistency

- Follow existing design patterns
- Keep layout clean and minimal
- Ensure responsiveness

### Scope Alignment

- Contributions should support the goal:

  > Building and improving GitHub README generators

## Getting Started

1. **Fork the repository**

2. **Clone your fork**

```bash
git clone https://github.com/BrandonBlkk/branreadme.git
```

3. **Create a branch**

```bash
git switch -c feat/your-feature-name
```

4. **Install dependencies**

```bash
npm install
```

5. **Run the project**

```bash
npm run dev
```

6. **Make changes and commit**

```bash
git commit -m "feat: your feature name"
```

7. **Push to your fork**

```bash
git push origin feat/your-feature-name
```

8. **Open a Pull Request**
